Act as a Senior Frontend Engineer. To maximize development speed for the Buildathon, we are abandoning the database-backed CMS for our Educator Blog. Instead, build a simple, local Markdown-based blog system in our Next.js app.

1. ARCHITECTURE

Create a new directory at the root of the project called _posts. This is where we will store our .md files.

Use gray-matter to parse frontmatter (title, date, author) and a lightweight markdown parser like remark / remark-html (or next-mdx-remote if you prefer) to render the content.

Create a utility function to read the file system (fs) and fetch all posts sorted by date.

2. FRONTEND: THE PUBLIC FEED (/build or /blog)

Fetch the list of markdown posts and display them in a clean, vertical feed.

Display the Title, Date, Author, and a short excerpt.

3. FRONTEND: THE POST PAGE (/blog/[slug])

Create a dynamic route that reads the specific .md file based on the URL slug and renders the HTML.

Use Tailwind's @tailwindcss/typography plugin (the prose class) to automatically style the markdown content.

4. STRICT DESIGN SYSTEM: MINIMALIST

Style: The blog feed and the individual posts must follow a strict minimalist aesthetic. Maximum whitespace, slate grays (bg-slate-50, text-slate-900), and pure off-white canvases.

Typography: Elegant, typography-focused. Light weights (font-light), precise hierarchy, and generous tracking for headers.

Borders: Use delicate 1-pixel slate-200 lines to separate posts in the feed. No heavy borders, drop shadows, or brutalist elements.

CRITICAL LAYOUT RULE: > Whenever you generate UI mockups, visual placeholders, or render this interface, always provide full-bleed digital layouts. Never include device frames, computer monitors, phone chassis, or physical background environments. The design should be perfectly flat, 2D, and fill the screen edge-to-edge.
