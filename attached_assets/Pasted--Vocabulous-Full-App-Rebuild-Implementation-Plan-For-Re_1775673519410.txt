### PHASE 2 — Real Authentication

**Goal**: Replace localStorage mock auth with DB-backed sessions.

> [!IMPORTANT]
> Auth currently uses hardcoded mock IDs (Teacher=1, Student=2). All database FK constraints fail because these IDs may not exist. This is the #1 blocker for all data features.

#### Backend: `artifacts/api-server/src/routes/users.ts`

Add `POST /api/auth/signup`:
- Accept `{ name, email, password, role }`
- Hash password with `bcryptjs`
- Insert into `users` table
- Return `{ id, name, email, role }` + set `userId` cookie (httpOnly)

Add `POST /api/auth/login`:
- Accept `{ email, password }`
- Verify hash, return user + set cookie

Add `GET /api/auth/me`:
- Read cookie, return current user or 401

Add `POST /api/auth/logout`:
- Clear cookie

Add `cookie-parser` and `express-session` (or stateless JWT via `jsonwebtoken`) to `artifacts/api-server`.

#### Frontend: `artifacts/srs-app/src/hooks/use-role.tsx`

Refactor `RoleProvider`:
- On mount, call `GET /api/auth/me` instead of reading localStorage
- Store `{ id, name, email, role }` in context
- Remove hardcoded IDs (1 and 2)
- Keep backward compat via `userId` from real DB response

#### Frontend: `artifacts/srs-app/src/pages/Signup.tsx`

Connect form `handleSubmit` to `POST /api/auth/signup` instead of `setTimeout` mock.

#### Frontend: `artifacts/srs-app/src/pages/Login.tsx`

Connect form to `POST /api/auth/login`.

### PHASE 8 — Student: Blurting Exercise Page

**Goal**: Free-recall writing practice graded by AI.

#### New File: `artifacts/srs-app/src/pages/student/StudentBlurting.tsx`

Route: `/student/blurting`

**Flow:**
1. Student picks a deck or tag to "blurt" from
2. A topic/prompt is displayed (e.g., "Write everything you know about Cellular Respiration")
3. A large textarea appears — student types freely
4. Timer (optional, user-set)
5. "Submit for Grading" calls `POST /api/ai/grade-blurt`
6. AI returns: Score (0–100), Rubric breakdown, Key concepts missed, Highlighted correct concepts
7. Results panel displays with color-coded feedback

**Backend endpoint**: `POST /api/ai/grade-blurt`
- Input: `{ prompt, studentResponse, deckId, rubric? }`
- Calls AI with rubric prompt
- Returns: `{ score, feedback, missedConcepts, correctConcepts }`
- Saves to `blurting_sessions` table

## 🤖 AI Integration Architecture

All AI endpoints should:
1. Use `OPENAI_API_KEY` or `GEMINI_API_KEY` from environment variables — never hardcoded
2. Have fallback graceful degradation if API is unavailable
3. Include rate limiting middleware
4. Cache results where appropriate (persona refreshes weekly, not on every request)

**Recommended AI library**: `openai` npm package (supports both OpenAI and compatible APIs)

```bash
pnpm --filter @workspace/api-server add openai
```

**AI endpoints to create** in `artifacts/api-server/src/routes/ai.ts`:
- `POST /api/ai/generate-cards` — card generation from terms + context
- `POST /api/ai/bulk-generate` — card generation from uploaded document text
- `POST /api/ai/grade-blurt` — score and grade a blurting exercise response
- `GET /api/ai/persona/:studentId` — generate/refresh learner persona analysis
